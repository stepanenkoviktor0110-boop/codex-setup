# Установка Codex CLI на macOS

Codex CLI — AI-помощник для программирования прямо в терминале от OpenAI.
Работает через подписку ChatGPT (Plus, Pro, Business, Edu).

---

## Перед началом: подготовка терминала

Открой **Терминал** (нажми `Cmd + Пробел`, напиши **Terminal** и нажми Enter).

Если при вставке команд из инструкции появляются странные символы вроде `^[[200~` — вставь и выполни эту строку один раз:

```bash
printf '\e[?2004l'
```

Это отключит режим, который ломает вставку. После этого все команды будут вставляться нормально.

---

## Шаг 1. Установи Node.js

Node.js — среда для запуска программ на JavaScript. Codex CLI работает на ней.

### Вариант A: Homebrew (если уже есть)

```bash
brew install node@22
```

### Вариант B: Установщик с сайта

1. Открой https://nodejs.org/
2. Нажми зелёную кнопку **"Download Node.js (LTS)"**
3. Скачай `.pkg` файл для macOS
4. Открой скачанный файл и следуй инструкции: **Продолжить → Продолжить → Установить** (введи пароль Mac, если попросит)

### Проверка

Открой **Терминал** (нажми `Cmd + Пробел`, напиши **Terminal** и нажми Enter) и введи:

```bash
node --version
```

Если видишь что-то вроде `v22.15.0` — всё ок. Нужна версия **20 или выше**.

## Шаг 2. Установи Codex CLI

В том же терминале:

```bash
npm install -g @openai/codex@latest
```

Подожди 10-30 секунд.

### Если появится ошибка `EACCES` (permission denied)

Это значит, что Node установлен системно (в `/usr/local`), и npm не может писать туда без root. **Не используй `sudo`** — это создаёт файлы, принадлежащие root, в домашней папке и всё ломает ещё сильнее.

Правильное решение — переключить npm на пользовательский префикс:

```bash
mkdir -p ~/.npm-global
npm config set prefix ~/.npm-global
echo 'export PATH="$HOME/.npm-global/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
npm install -g @openai/codex@latest
```

**Автоматический скрипт `scripts/setup-codex.sh` делает всё это сам** — включая починку прав на кэш `~/.npm` и удаление старых системных установок, если ты раньше пробовал `sudo npm install`.

### Альтернативный способ — Homebrew

```bash
brew install codex
```

## Шаг 3. Авторизуйся

Codex CLI работает через подписку ChatGPT. Нужна активная подписка:
- **ChatGPT Plus** — $20/мес
- **ChatGPT Pro** — $200/мес
- **Business / Edu / Enterprise** — тоже подходят

Запусти авторизацию:

```bash
codex login
```

Откроется браузер — войди в свой аккаунт OpenAI. После входа терминал покажет, что авторизация прошла.

Если браузер не открылся автоматически:

```bash
codex login --device-auth
```

Появится код — открой ссылку в браузере вручную и введи этот код.

## Шаг 4. Запусти

```bash
codex
```

Теперь можно писать запросы прямо в терминале:

```
напиши функцию сортировки на Python
```

Codex ответит, напишет код и может создать файлы в твоей папке.

## Частые проблемы

### "codex: command not found"

Закрой терминал и открой заново. Если не помогло:

```bash
npx @openai/codex@latest
```

### Ошибка прав доступа (EACCES) при установке

**Не используй `sudo npm install`** — это создаст файлы, принадлежащие root, в твоей домашней папке и сломает все последующие установки без sudo.

Правильно: переключить npm на пользовательский префикс (см. выше) или запустить `bash scripts/setup-codex.sh` — он делает всё автоматически.

Если `sudo npm install` уже был сделан и всё сломалось, почини одной командой:

```bash
sudo chown -R $(id -u):$(id -g) ~/.npm ~/.npm-global 2>/dev/null
sudo rm -rf /usr/local/lib/node_modules/@openai /usr/local/lib/node_modules/codex-cli /usr/local/bin/codex
```

Потом повтори шаг 2 без `sudo`.

### Старая версия Node.js

Если `node --version` показывает версию ниже 20 — обнови:

```bash
brew install node@22
```

Или скачай новую версию с https://nodejs.org/

### Авторизация не проходит

1. Проверь, что подписка активна: https://chatgpt.com/settings
2. Попробуй device auth: `codex login --device-auth`
3. Если токен протух — просто запусти `codex login` заново
