# 04. Безопасность

## Проблема

AI-агент выполняет команды в терминале. Без ограничений он может случайно:
- Удалить системные файлы (`rm -rf /`)
- Отформатировать диск
- Скачать и выполнить вредоносный скрипт (`curl ... | bash`)
- Перезагрузить компьютер

## Решение: защитные хуки

Скрипт устанавливает хуки — фильтры, которые проверяют каждую команду перед выполнением.

### Что блокируется

- `rm -rf /`, `rm -rf ~`, удаление системных папок (`/System`, `/Library`, `/usr`)
- `sudo rm -rf *`
- Форматирование (`mkfs.*`, `dd if=... of=/dev/...`)
- Pipe в shell (`curl ... | bash`, `wget ... | sh`)
- Перезагрузка (`shutdown`, `reboot`, `halt`)
- Fork-бомба (`:(){:|:&};:`)
- `chmod 777 /`
- Завершение системных процессов (`killall Finder/Dock`)

### Режим работы

**Full-auto** — агент выполняет команды без подтверждения, но опасные команды блокируются хуками. Это баланс между удобством и безопасностью.

## Файлы

- `~/.codex/hooks.json` — конфигурация хуков (какие события перехватывать)
- `~/.codex/hooks/block-dangerous.sh` — скрипт проверки команд
- `~/.codex/config.toml` — `codex_hooks = true` (включает хуки), `approval_policy = "full-auto"`

## Как добавить свой паттерн

Открой `~/.codex/hooks/block-dangerous.sh` и добавь паттерн в массив `BLOCKED_PATTERNS`:

```bash
BLOCKED_PATTERNS=(
    ...
    'твой_новый_паттерн'
)
```

## Как отключить хуки

В `~/.codex/config.toml`:

```toml
[features]
codex_hooks = false
```
