# Настройка Codex CLI

> Один скрипт — и Codex CLI от OpenAI полностью настроен на macOS: установка, авторизация, профиль, безопасность, мониторинг процессов и методология AI-разработки.

## О проекте

Codex CLI устанавливается за минуту, но без настройки работает без ограничений — может удалить файлы, засорить систему зависшими процессами, а эффективные паттерны работы с AI-агентом приходится выстраивать с нуля.

Этот проект решает все три проблемы: setup-скрипт, защитные хуки, Agent Cleaner и адаптированная методология — всё в одной команде.

## Что устанавливается

- **Node.js v22** — без sudo, через Homebrew или напрямую
- **Codex CLI** (`@openai/codex`) — AI-агент в терминале
- **Авторизация OAuth** — вход через аккаунт ChatGPT (Plus/Pro подписка)
- **Профиль пользователя** — 6 вопросов, ответы в `~/.agents/AGENTS.md`
- **Защитные хуки** — блокировка опасных команд (rm -rf, curl|bash и т.д.)
- **Agent Cleaner** — автоматическая очистка зависших процессов (launchd)
- **Методология** — skills и agents из ai-dev-methodology-codex

## Структура

- `docs/` — пошаговые гайды (установка, профиль, безопасность, cleaner, методология)
- `scripts/` — автоматизация (setup-codex.sh, agent_cleaner.py, install-hooks.sh)
- `configs/` — шаблоны конфигов (config.toml, hooks.json, launchd plist)
- `skills/` — навыки для Codex (копируются из ai-dev-methodology-codex)
- `agents/` — агенты (копируются из ai-dev-methodology-codex)

## Быстрый старт

Открой Терминал на Mac и вставь:

```bash
curl -L https://github.com/stepanenkoviktor0110-boop/codex-setup/archive/refs/heads/main.tar.gz | tar xz
cd codex-setup-main
bash scripts/setup-codex.sh
```

Если нет `curl` (маловероятно на macOS), скачай архив вручную со страницы релизов.

## Требования

- macOS 12+ (Monterey и новее)
- Терминал (Terminal.app или iTerm2)
- Интернет-соединение
- Подписка ChatGPT Plus ($20/мес) или Pro ($200/мес)
- **VPN с выходом в поддерживаемую страну** (US/ЕС и т.п.), если ты в РФ/Беларуси — OpenAI блокирует доступ по IP. Без VPN шаг авторизации упадёт с `Country, region, or territory not supported`.

## Повторный запуск

Скрипт идемпотентный — повторный запуск не ломает уже настроенное. Пропускает шаги, которые уже выполнены, и обновляет то, что изменилось.

## Проблемы

Если при вставке текста в терминал появляются лишние символы (`~0` или `~1`), выполни перед запуском:

```bash
printf '\e[?2004l'
```
